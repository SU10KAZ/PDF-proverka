## Что именно извлекать

### Кабели (cables)

Для каждой кабельной линии которую видишь:

```json
{
  "line_id": "М-1.1",           // ОБЯЗАТЕЛЬНО — точно как в проекте (М-1.1, М-2.4, ...)
  "mark": "ППГнг(А)-HF",        // марка кабеля
  "phase_count": 4,             // количество фазных жил
  "phase_section_mm2": 185,     // сечение фазной жилы
  "pe_or_n_count": 1,           // количество PE/N/нулевых жил
  "pe_or_n_section_mm2": 185,   // сечение PE/N жилы
  "pe_or_n_type": "PE",         // "N" | "PE" | "PEN" | "не_указано" — как в проекте
  "source_panel": "РП1 ввод №1",
  "destination": "ВРУ-1",
  "length_m": null,             // если указана длина
  "evidence": {
    "raw_quote": "ППГнг(А)-HF 2×(4×(1×185))+1×(1×185)",  // точная цитата с чертежа
    "confidence": 0.9,
    "visual_location": "верхняя часть схемы РП1"
  },
  "alternate_readings": []       // если есть два варианта — указать второй
}
```

**КРИТИЧНО — как читать кабели:**

Формат: `{марка} {фазы}×(1×{сечение_фазы})+{кол-во_PE}×(1×{сечение_PE})`

Примеры:
- `ППГнг(А)-HF 4×(1×185)+1×(1×185)` → 4 фазы 185мм² + 1 PE 185мм²
- `ППГнг(А)-HF 2×4×(1×185)+2×(1×185)` → 8 фаз 185мм² + 2 PE 185мм² (две параллельные линии)
- `4×(1×240)+(1×120)` → 4 фазы 240мм² + 1 PE 120мм²

**Если есть две параллельные линии** (2× в начале) — это важно! Записать правильно `phase_count` (сумма) и `pe_or_n_count` (сумма).

**Если рядом с цифрой написано "PE", "N", "нулевой", "защитный" — записать в `pe_or_n_type`.**

### Автоматы (breakers)

```json
{
  "designation": "QF1.7",           // позиционное обозначение (QF1.1, QF2.4, ...)
  "model": "ВА-302",                // марка АВ
  "current_rating_a": 50,           // номинальный ток
  "trip_type": "термомагнитный",    // "термомагнитный" | "электронный" | "не_указано"
  "location": "РП1",                // где установлен (ГРЩ, ЩСН, РП1, РП2)
  "protects_line": "ЩСН",           // что защищает (если видно)
  "evidence": {
    "raw_quote": "QF1.7 ВА-302 50А",
    "confidence": 0.85
  }
}
```

**Правила определения типа расцепителя:**
- Марка ВА-302, ВА-305 (модульные) → `термомагнитный`
- Марка ВА-332А, ВА-333А, ВА-335А (стационарные литого корпуса) → `термомагнитный` если не указано иное
- Если рядом написано "электронный", "электр.", "electr" → `электронный`
- Если рядом написано "термомагнитный", "терм.маг." → `термомагнитный`
- В остальных случаях → `не_указано`

### Щиты и панели (panels)

```json
{
  "id": "ЩУ-2/Т",                    // идентификатор щита
  "type": "шкаф_учёта",              // тип
  "description": "Шкаф учета на 12 счетчиков трансф.вкл.",  // полное описание если видно
  "source_feed": "ВРУ-1",            // откуда питание (если видно)
  "count_in_spec": null,             // если это строка спецификации — количество
  "evidence": {
    "raw_quote": "ЩУ-2/Т — Шкаф учета на 12 счетчиков",
    "confidence": 0.9,
    "context": "spec_table_row"       // schema_label | spec_row | plan_label | note
  }
}
```

### Примечания (notes)

Это самое важное для класса 4. Извлекай **все** текстовые примечания на блоке:

```json
{
  "text": "АВ отходящих линий ГРЩ применить с электронными расцепителями, обладающими селективной задержкой срабатывания от КЗ",
  "scope_hint": "отходящие АВ ГРЩ",  // к чему относится (если можешь определить)
  "category": "requirement",          // "requirement" | "reference" | "instruction" | "general"
  "evidence": {
    "raw_quote": "...",
    "location": "под таблицей автоматов"
  }
}
```

**КРИТИЧНО для примечаний:** если видишь текст который декларирует **требование**
к оборудованию или монтажу — извлеки его **дословно**. Это не просто примечание,
это **правило которое потом будет проверяться**.

Примеры требований:
- "применить АВ с электронными расцепителями"
- "кабели с индексом FR"
- "не менее 10% резервных АВ"
- "в огнестойком исполнении EI150"
- "ТТ класс точности не хуже 0.5S"

## Выходной JSON Schema

```json
{
  "batch_id": {BATCH_ID},
  "project_id": "{PROJECT_ID}",
  "timestamp": "<ISO datetime>",
  "block_extractions": [
    {
      "block_id": "7CWN-7HQG-JPW",
      "page": 14,
      "sheet": "Лист 10",
      "sheet_type": "schema|node|plan|spec_table|notes|stamp",
      "block_summary": "Узел раскладки 1: сечение кабелей ВРУ-1, ВРУ-2, ВРУ-3 через строительную конструкцию",
      "entities": {
        "cables": [ ... ],
        "breakers": [ ... ],
        "panels": [ ... ],
        "notes": [ ... ],
        "current_transformers": [ ... ]
      },
      "unreadables": [ ... ]
    }
  ]
}
```

### Правила сборки

1. **Каждый блок** = одна запись в `block_extractions[]`
2. **Сущности внутри блока** — только те что видишь НА ЭТОМ блоке (не на соседних)
3. **Если сущности нет** на блоке → пустой массив `[]`
4. **Title блоки и штампы** → `sheet_type: "stamp"`, `entities: {}`, `block_summary: "Штамп"`
5. **Пустые/нечитаемые блоки** → всё равно возвращать с `unreadables[]`

### Поля которых нет на блоке

Если не видишь значения — пиши `null` (не пропускай поле, не придумывай):
```json
{"line_id": "М-1.1", "phase_section_mm2": null, ...}
```

Если видишь два возможных значения — возвращай оба:
```json
{
  "line_id": "М-1.1",
  "phase_section_mm2": 185,
  "alternate_readings": [
    {"phase_section_mm2": 150, "confidence": 0.3, "reason": "возможно плохо видно"}
  ]
}
```

### ПОДРОБНО для однолинейных схем

Однолинейные схемы ГРЩ/РП/ВРУ — самые важные блоки. Для них требования:

1. **Прочитать ВСЕ марки кабелей** — вертикальный текст вдоль линий
2. **Прочитать ВСЕ номиналы автоматов** — около условных обозначений
3. **Прочитать ВСЕ позиционные обозначения** — QF, TA, KM, QS
4. **Прочитать ВСЕ обозначения линий** — М-1.1, М-2.3 и т.д.
5. **Прочитать ВСЕ обозначения щитов** — ВРУ-1, ЩСН, ЩУ-2/Т

Если текст мелкий или повёрнут — всё равно читать. Блоки рендерятся в 2500px для этой цели.

Если точно не видно — `"confidence": 0.4` и отметка в `unreadables`.
