## Уровни критичности замечаний

Для ВСЕХ дисциплин используются одинаковые 5 уровней:

| Уровень | Описание | Примеры |
|---|---|---|
| **КРИТИЧЕСКОЕ** | Нарушение обязательных норм. Нельзя строить / эксплуатировать | Нарушение ПУЭ/ГОСТ/СП обязательного применения, угроза жизни |
| **ЭКОНОМИЧЕСКОЕ** | Финансовые потери, перерасход, пересортица | Завышенные сечения, лишнее оборудование, ошибки в объёмах |
| **ЭКСПЛУАТАЦИОННОЕ** | Будущие проблемы при эксплуатации | Неудобный доступ, отсутствие резервирования, нестандартные решения |
| **РЕКОМЕНДАТЕЛЬНОЕ** | Опечатки, мелкие несоответствия, улучшения | Опечатки в марках, несовпадение нумерации, оформление |
| **ПРОВЕРИТЬ ПО СМЕЖНЫМ** | Требует информации из других разделов проекта | Мощности из ОВ, нагрузки из ВК, архитектурные решения |

severity в JSON: `"КРИТИЧЕСКОЕ"`, `"ЭКОНОМИЧЕСКОЕ"`, `"ЭКСПЛУАТАЦИОННОЕ"`, `"РЕКОМЕНДАТЕЛЬНОЕ"`, `"ПРОВЕРИТЬ ПО СМЕЖНЫМ"`
