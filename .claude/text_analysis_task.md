## Задача

### Этап 1: Анализ текстовых данных

Прочитай MD-файл ПОЛНОСТЬЮ. Извлеки:

1. **Параметры проекта** (`project_params`):
   - Тип объекта, этажность, площади
   - Расчётные нагрузки, мощности, расходы
   - Основное оборудование, марки, типоразмеры

2. **Нормативные ссылки** (`normative_refs_found`):
   - Все упомянутые СП, ГОСТ, ПУЭ, ФЗ
   - Проверить актуальность каждой нормы по файлу нормативной базы

3. **Предварительные замечания** (`text_findings`, нумерация T-001, T-002...):
   - Несоответствия в расчётах
   - Устаревшие нормативные ссылки
   - Противоречия между разделами
   - Пропущенные обязательные данные

4. **Арифметическая сверка таблиц** (ОБЯЗАТЕЛЬНО):
   - Пересчитай суммы строк и столбцов в КАЖДОЙ таблице нагрузок
   - Пересчитай расчётные величины по формулам дисциплины (см. чек-лист ниже)
   - Проверь площади и мощности на реалистичность
   - Сверь суммарные мощности/расходы оборудования с каталожными данными

5. **Перекрёстная сверка** (ОБЯЗАТЕЛЬНО):
   - ПЗ vs таблицы нагрузок — совпадают ли числа?
   - Спецификация vs текст ПЗ — марки, количества, типоразмеры
   - Расхождения между разными местами документа → замечание
   - Стандартные типоразмеры — проверь что указанные размеры существуют в ГОСТ-сортаменте

6. **Диапазоны и характеристики оборудования** (ОБЯЗАТЕЛЬНО):
   - Диапазон измерительных приборов должен соответствовать рабочим параметрам
   - Типоразмеры — существуют ли в каталогах?
   - Мощности — совпадают ли с каталожными?

7. **Сверка спецификации с [IMAGE]** (ОБЯЗАТЕЛЬНО):
   - Оборудование на чертеже, которого нет в спецификации → замечание
   - Позиция в спецификации, которая не встречается ни на одном чертеже → замечание

## Нормативная точность (norm_quote + norm_confidence)

Для КАЖДОГО замечания `text_findings[]` с полем `norm`:
- **`norm_quote`** — точная цитата текста пункта нормы (1-2 предложения). `null` если не помнишь точно.
- **`norm_confidence`** — уверенность 0.0–1.0 что указанный пункт содержит это требование.
  - 1.0 = точно знаешь, 0.7–0.9 = помнишь суть, < 0.7 = не уверен.
  - При confidence < 0.8 верификатор проверит цитату через WebSearch.

## Правила

1. Прочитай MD-файл ПОЛНОСТЬЮ — не пропускай секции
2. `text_findings[]` — только по текстовым данным (не по чертежам)
3. `blocks_for_review[]` — заполнить для каждого блока из index.json
4. `severity` — ТОЛЬКО одно из 5 значений
5. Пиши JSON через Write — НЕ выводи в чат
6. После записи выведи краткий итог
