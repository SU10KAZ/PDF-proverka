## ЭТАП 02.5 — АНАЛИЗ ПРОБЕЛОВ (GAP ANALYSIS)

Проверить, достаточно ли данных:

   с чертежей, которые НЕ были проанализированы?

**Если пробелов нет** — записать `gap_analysis: null`.

## ЭТАП 03 — СВОД ЗАМЕЧАНИЙ

Объединить preliminary_findings из ОБОИХ этапов (01 + 02).

### Правила объединения
1. **Дедупликация:** если одно замечание найдено и в тексте, и на чертеже — оставить ОДНУ запись с более полным описанием
2. **Повышение severity:** если текстовое замечание подтверждено чертежом — severity может повыситься
3. **Понижение severity:** если текстовое подозрение НЕ подтвердилось на чертеже — убрать или понизить
4. **Перенумерация:** финальные ID: F-001, F-002, ...

### Формат findings
- `severity`: ТОЛЬКО одно из 5 — КРИТИЧЕСКОЕ, ЭКОНОМИЧЕСКОЕ, ЭКСПЛУАТАЦИОННОЕ, РЕКОМЕНДАТЕЛЬНОЕ, ПРОВЕРИТЬ ПО СМЕЖНЫМ
- `problem`: краткая суть (1-2 строки)
- `description`: развёрнутое описание с числовыми данными
- `norm`: конкретный документ + пункт (со статусом актуальности)
- `solution`: конкретное действие по исправлению
- `risk`: последствия если не исправить

## ЭТАП 04 — ФИНАЛЬНЫЙ ОТЧЁТ

ЧИТАТЬ ТОЛЬКО: `{OUTPUT_PATH}/03_findings.json`

ЗАПИСАТЬ: `{OUTPUT_PATH}/audit_results_YYYYMMDD.md`

Формат Markdown:
```
# Аудит проекта: <project_id>
Дата: <дата>
Режим: Smart Parallel Audit (параллельный интеллектуальный аудит)
Проанализировано: X из Y чертёжных страниц
Итого замечаний: N (Критично: N, Экономич.: N, Эксплуат.: N, Рекоменд.: N)

| № | Критичность | Лист/стр | Замечание | Норма | Рекомендация |
| F-001 | КРИТИЧНО | ... | ... | ... | ... |

## Пропущенные страницы (LOW/SKIP)
| Стр. | Тип | Причина пропуска |

## Резюме (3-5 предложений)
```

## Правила

1. Пиши JSON через инструмент Write — НЕ выводи в чат
2. ID замечаний: F-001, F-002, ... (сквозная нумерация)
3. Формат даты: ISO 8601
4. При ссылке на норму — указывай статус (действует/заменён/отменён)
5. Если gap_analysis требует дополнительных страниц — запиши их в JSON,
   конвейер обработает автоматически
6. После записи JSON выведи краткий итог: сколько замечаний, по критичности
