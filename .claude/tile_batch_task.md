## Формирование page_summaries (ОБЯЗАТЕЛЬНО)

После анализа всех тайлов — для КАЖДОЙ страницы в пакете сформировать запись `page_summaries`.
Это **синтез** информации со всех тайлов страницы в единое целое.

### Порядок склейки текста
1. Сортировка тайлов: по строкам, внутри строки по столбцам (r1c1 → r1c2 → r2c1 → r2c2)
2. Восстановить разрезанные предложения и числа на границах тайлов
3. Убрать дубли из зон перекрытия (~5%)
4. Результат — `full_text_content`: весь читаемый текст страницы единым потоком

### Определение типа листа (sheet_type)
Используй СТРОГО один из следующих значений:

{DISCIPLINE_DRAWING_TYPES}

### Поля is_partial и rows_covered
- Если пакет содержит ВСЕ тайлы страницы → `is_partial: false`, `rows_covered` = все ряды
- Если пакет содержит только ЧАСТЬ рядов (большая страница разбита) → `is_partial: true`, `rows_covered` = только ряды в этом пакете

### Поле key_values
Самые важные значения со страницы: номиналы АВ, марки кабелей, мощности, токи. Формат как на чертеже.

### Поле findings_on_page
ID замечаний (G-NNN) найденных на этой странице. Должны совпадать с ID из `preliminary_findings`.

## Правила

1. **Прочитай КАЖДЫЙ тайл** — не пропускай ни один
2. Для каждого тайла ОБЯЗАТЕЛЬНО запиши summary и key_values_read
3. Если тайл пустой или содержит только штамп — запиши "Штамп / служебная информация"
4. findings может быть пустым [] если замечаний нет — это нормально
5. Нумерация: G-001, G-002, ... (в пределах этого пакета, при слиянии перенумеруется)
6. severity — ТОЛЬКО одно из 5 значений: КРИТИЧЕСКОЕ, ЭКОНОМИЧЕСКОЕ, ЭКСПЛУАТАЦИОННОЕ, РЕКОМЕНДАТЕЛЬНОЕ, ПРОВЕРИТЬ ПО СМЕЖНЫМ
7. Пиши JSON через инструмент Write — НЕ выводи в чат
8. После записи JSON выведи краткий итог: сколько тайлов, сколько находок
